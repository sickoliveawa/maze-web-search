\begin{finding}[``均值降/峰值升'' 现象]
族数上限从 mf=1 增至 mf=8 时,均值从 0.6984 降至 0.6306,
但峰值却由 mf=1 的 0.7999 升至 mf=8 的 \textbf{0.8233}。
多族联合优化增加了 ES 搜索空间,拖累均值;
但 mf=8 拥有全场最高分,优势在 ``尾部上界更高'',而非 ``整体均值更高''。
\end{finding}

\subsection{Mask × Family Cap 消融分析}
\label{sec:mask-fam-heatmap}

表~\ref{tab:mask-mean-v13} 的数据将失败模式清晰分为两类,机制不同:

\noindent\textbf{(1) chebyshev-4: 搜索空间过大,预算不足。}
其邻域严格包含 chebyshev-2,理论上 chebyshev-2 的最优解是其子集,
但 chebyshev-4 得分低 0.59 分。
80 位 cells mask 占据 $2^{80}$ 种取值,叠加 birth ($2^9$) + survive ($2^9$)
+ priority ($2^4$) 后,单族总搜索空间 $=2^{103} = 2^{80}\times 2^{23}$;
$200\!\times\!500$ 预算 (约 $10^5$ 次评估) 对 $2^{103}$ 子空间过于稀疏,
ES 在大量 ``无梯度平台'' 上原地踏步。
\textbf{这一类失败来自预算约束,不来自模板表达能力。}

\noindent\textbf{(2) manhattan-1: 邻域信息不足,无法涌现走廊结构。}
manhattan-1 是 4-邻域 (von Neumann),每个元胞每步最多感知 4 个邻居,
而走廊结构 (1-cell 宽、长程连通) 需要 ``沿走廊方向存活 + 左右墙存活''
的二阶条件,4 邻域只能 ``凑巧'' 出现,无法稳定涌现。
与 manhattan-2 (8 邻域) 差 0.358 分,是直接编码对角线存活条件的能力鸿沟。
\textbf{这一类失败来自模板本身的信息瓶颈,不来自预算。}

\subsection{小范围 vs 大范围对比小结}
\label{sec:sweep-comparison}

小范围 sweep 122 个 OK run 提供完整的 mask template 与 family cap 消融
数据 (§5.3);大范围 sweep 的 5/5 run 提供 ``同配置,大网格'' 的 scale-up
证据 (§5.2)。
两者的对比要点:

\begin{itemize}[itemsep=0pt, topsep=2pt, leftmargin=*]
  \item 配置:大范围只测了 manhattan-2/mf=8 一种配置,小范围测了
        $8\!\times\!4=32$ 种 (mask, mf) 组合。
  \item 网格:大范围 $500\!\times\!2000$ = 1M 像素,小范围 $40\!\times\!60$ = 2.4K
        像素,大范围约高 170 倍面积。
  \item 结果:大范围 5-run 最高 0.8195,小范围单点最高 0.8233,
        差距仅 0.0037 (§5.2 finding)。
  \item 解释:manhattan-2/mf=8 在 $40\!\times\!60$ 网格上已接近该配置的
        理论上界 (§6.3);大范围 sweep 用于验证 ``峰值不显著下降'' 而非
        ``寻找新高分''。
\end{itemize}
