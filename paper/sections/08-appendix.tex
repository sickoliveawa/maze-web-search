\subsection{FamilyMask 染色体判定}

设第 $i$ 族 $\mathcal{F}_i = (\text{cells}_i, \text{birth}_i, \text{survive}_i, \text{priority}_i)$,
全局 $\texttt{active}$ 集合 $A \subseteq \{0, \ldots, 15\}$。
对元胞 $(x,y)$ 在时刻 $t$ 的更新:
\begin{align}
  n_i(x,y) &=
  \sum_{(\Delta x, \Delta y) \in \text{cells}_i}
  \mathbb{1}\!\left[G_t(x{+}\Delta x, y{+}\Delta y) > 0\right] \\
  \tilde{n}(x,y) &= n_{i^*}(x,y), \quad
  i^* = \min\bigl\{\,i \in A
  : \big(G_t(x,y) = 0 \wedge n_i(x,y) \in \text{birth}_i\big)
  \vee \big(G_t(x,y) > 0 \wedge n_i(x,y) \in \text{survive}_i\big)\,\bigr\} \\
  G_{t+1}(x,y) &=
  \begin{cases}
    1, & G_t(x,y) = 0 \wedge n_{i^*}(x,y) \in \text{birth}_{i^*} \\
    1, & G_t(x,y) > 0 \wedge n_{i^*}(x,y) \in \text{survive}_{i^*} \\
    0, & \text{otherwise}
  \end{cases}
\end{align}
若 $i^*$ 不存在, $G_{t+1}(x,y) = \texttt{defaultState} = 0$。

\subsection{maze\_quality 子度量推导}

记 $V = \{v : G(v) = 1\}$, $|V| = N$, $\deg(v) = \#\{u \sim v : G(u) > 0\}$,
$L_{\max}$ 为 (4-邻) 最长简单路径,
$C_{\max}$ 为最大 4-连通分量,
$a_{\text{outer}}$ 为四条边上的活格子总数 (4 角去重), 
$H(\cdot)$ 为 Shannon 熵 (自然对数除以 $\ln 2$ 给出比特单位)。

\begin{align}
  M_b &=
  \frac{\mathrm{clip}\!\left(\#\{v \in V : \deg(v) = 2\} / N - 0.4,\, 0,\, 0.5\right)}{0.5} \\
  M_s &=
  \begin{cases}
    \min\!\left(1,\, \dfrac{L_{\max}/N}{0.3}\right), & \dfrac{L_{\max}}{N} \le 0.5 \\[4pt]
    \max\!\left(0,\, 1 - \dfrac{L_{\max}/N - 0.5}{0.5}\right), & \dfrac{L_{\max}}{N} > 0.5
  \end{cases}
  M_j &= \exp\!\left[-\left(\frac{\#\{v : \deg(v) \ge 3\} / N - 0.10}{0.10}\right)^{2}\right] \\
  M_c &= \mathrm{clip}\!\left(\frac{|C_{\max}|}{0.8 N},\, 0,\, 1\right) \\
  M_{\text{bnd}} &= \mathrm{clip}\!\left(1 - \frac{a_{\text{outer}} - 4}{76},\, 0,\, 1\right) \\
  M_p &= \frac{H(\text{2x2 patch})}{\log_2 16} \\
  M_a &=
  \begin{cases}
    0.5, & \max \mathrm{match}(G, G^{\mathrm{flip}}) < 0.55 \\
    1.0, & 0.55 \le \max \mathrm{match} < 0.85 \\
    1 - \dfrac{\max \mathrm{match} - 0.85}{0.15}, & \max \mathrm{match} \ge 0.85
  \end{cases} \\
  M_t &= \frac{H(\text{2x2 pair})}{\log_2 256} \\
  M_{\text{wall\_gate}} &=
  \begin{cases}
    0 & M_{\text{wall}} \in \{0, 1\} \\
    M_{\text{wall}} / 0.4 & M_{\text{wall}} \in [0, 0.4] \\
    1.0 & M_{\text{wall}} \in [0.4, 0.6] \\
    (1 - M_{\text{wall}}) / 0.4 & M_{\text{wall}} \in [0.6, 1]
  \end{cases}
\end{align}

其中 $M_{\text{wall}} = \#\{v : G(v) = 0\} / (W H)$ 为墙比。代码实现见
\texttt{src/metrics/maze\_quality.js}。

\subsection{Bellot 2021 $F = \nu/\delta$ 对照}

Bellot 2021 提出 $F(M) = \nu(M) / \delta(M)$ (Bellot 2021 §3.4),
本文对照基准使用 source \texttt{src/gpu/bellot\_metrics.js::bellotF} 真实实现,
其中 $\nu$ = 非显著墙计数 (死端邻接类),\quad
$\delta$ = twistiness proxy (diameter / diagonal)。
\textbf{注意:} \texttt{scripts/full\_mq\_v2.mjs} 中的简易
$F = (1\text{-roadFrac}) / \max(0.5,\, d/200)$ 不再使用,
其数值与 Bellot 2021 实际公式不一致。

\section{ckpt 与 ndjson 时序错位的证据}
\label{app:repro-logs}

\noindent\textbf{本文数值证据的 ``原始唯一来源'' 是
\tt{sweep\_2026\_07\_04/results.ndjson},而非 ckpt 文件。}
ndjson 中 panel (b) (chebyshev-1/mf=8/seed=333) 的完整字段如下:

\begin{verbatim}
{
  "mask": "chebyshev-1",
  "maxFam": 8,
  "seed": 333,
  "pop": 200,
  "gens": 500,
  "gridW": 40,
  "gridH": 60,
  "best": 0.809539,
  "ts": "2026-07-04T17:29:13.531939",
  "status": "OK"
}
\end{verbatim}

\noindent\textbf{ckpt 文件的时间戳不一致。}对应 ckpt 文件
\tt{sweep\_chebyshev-1\_mf8\_s333.json} 的保存时间为
\tt{savedAt} = \tt{2026-07-06T13:11:35.281Z},比训练时间戳晚约 44 小时 (43.71 小时, 早期 v1.0 草稿曾误写 49h, 已更正);
\emph{mtime} (文件系统戳) 也指向该时刻。多次访问 (file stat) 期间
\tt{savedAt} 与 \emph{mtime} 互相对齐到 07-06 13:11,说明该文件至少被
覆写过一次。

\noindent\textbf{同 batch 名下存在 \tt{panel\_b\_test.json} (gen=50)。}
对 ckpt 目录做 file-search 可见:

\begin{verbatim}
$ find ckpt/ -name '*chebyshev-1_mf8_s333*'
sweep_chebyshev-1_mf8_s333.json           <-- 44h 后被覆写 (43.71h 实测)
sweep_chebyshev-1_mf8_s333.json.bak_*     <-- 备份
panel_b_test.json                         <-- gen=50 测试写
\end{verbatim}

\noindent\textbf{\tt{panel\_b\_test.json} 的存在本身就是 dispatcher
竞态的证据:} 同 batch 名 (\tt{chebyshev-1\_mf8\_s333}) 下,gen=500 写
文件尚未触发时,gen=50 测试写就已经写入同目录的另一个文件。
这是源代码中 \tt{src/ckpt.js:35 saveCheckpoint} 用 batch 名为文件名
+ ``overwrites when same name'' \ 设计带来的固有问题 — 一旦多次写操作用同
batch 但不同 gen,并发覆盖就会发生。本文不能修改 ``在 Paper 写出过程
中已验证正确的部分'',因此该设计缺陷作为限制在 §6.4 与 §5.5 中被记录,
而非尝试回滚/修复。

\noindent\textbf{因此}:ckpt 位串作为 ``最优个体的可视快照''可读,
作为 ``可重放的最优个体''不可保证;Paper 中所有具体数值一律以 ndjson
为来源,ckpt 仅充当 layout/architecture 图示(图 5.4)。

\noindent\textbf{关键事实:}
\begin{itemize}\itemsep0pt\topsep2pt
  \item 仅 panel (e) ({\tt chebyshev-4/mf=1/s=111}) 在当前浏览器 GPU + mq 下
        {\bf 可重放} (live\_total $=$ saved\_bestScore $=$ $0.4244$,delta $=0$)。
  \item panel (b) 的 $\text{gap}=+43.71$ 小时是
        {\bf 唯一} 的 Mode A (race-overwrite) 案例,见前述分析。
  \item 其它 5 个 panel (a,c,d,e,f) 的 $\text{gap}=-8$ 小时是正常的 pipeline ordering
        (ndjson 末尾日志比 ckpt 写盘晚 $\sim\!8$ h),无 race-overwrite。
  \item 这 5 个 panel 都在当前浏览器 GPU 下 {\tt live\_total=$0$},但
        {\tt M\_WR\_gate} (位串派生) 仍正确,只有 {\tt M\_boundary}、{\tt M\_topology}
        (grid 派生) 塌缩为 0 — 与 {\tt src/metrics/maze\_quality.js}
        2026-07-06T15:25 修改后的版本行为一致 (Mode B: scoring drift)。
  \item verifier pipeline (cdp + canvas readback + mq eval) 经 panel (e) MATCH
        反向证明健全,排除 Mode C。
\end{itemize}

\noindent\textbf{panel (e) 是唯一的例外。}其 {\tt bestScore=$0.4244$} 是
``纯 fail'' 区段 (chebyshev-4 整体均值 $\sim\! 0.157$),其 0.42 量级
刚好在 mq 当前的 {\tt wall\_ratio} gate 边界内能通过;而 (a)(b)(c)(d)(f)
的 $0.7+$ 量级规则经过新 mq 的 gate / connectivity 重算后结构分量为 0。
这支持 {\bf Mode B (scoring drift)} 归因,而非位串错或 GPU pipeline 错。

\noindent\textbf{§5.3 fig 5.4 panel (c)(f) 的 {\tt mf} 选取。}fig 5.4 中
panel (c) ({\tt chebyshev-2}, score $0.8080$) 与 panel (f) ({\tt manhattan-1},
score $0.4240$) 实际使用的是 {\tt mf=2} 的 ckpt,而非 {\tt mf=8}:ndjson
数据显示 {\tt chebyshev-2} 的全场最佳 ($0.8080$) 出现在 {\tt mf=2/s=333} 而非
{\tt mf=8/s=333} (后者仅 $0.7116$);{\tt manhattan-1} 的全场最佳
($0.4360$) 出现在 {\tt mf=4/s=111},而 fig 5.4 panel (f) 选择的是 {\tt mf=2/s=444}
($0.4240$),作为 ``贴近均值 $0.4205$ 的 fail-mode 范例''。
fig 5.4 caption 中已注明 ``{\tt s=333}''、``{\tt s=444}'',但未注明
{\tt mf=2} 是因为该数值在每个 mask 内是 ``{\tt best per mask}'' 而非
``{\tt best per mf=8}'',§5.3 prose 现统一为 ``best of mask''(而非 ``mf=8 唯一最优'')。
